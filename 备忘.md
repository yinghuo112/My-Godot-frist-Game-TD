## 自动地图生产

| 方案 | 实践次数 | 优点 | 缺点 | 选型 |
|------|---------|------|------|------|
| TileMapDual | 1 | 路径可控 / 瓦片资源少(47→15) / 与数据平衡无缝衔接 | 手动绘制，多图稍慢 | **当前推荐** |
| WFC 插件 | 0 | 批量生成快 / 风格统一 | 路径不可控 / 需人工验收 / 学成本高 | 备选 |

### 多地图基础设施（已完成）
- `核心/slot_data.gd` — 槽位数据资源（position / difficulty）
- `核心/map_data.gd` — 地图数据资源（map_id / slots）
- `核心/map_manager.gd` — `load_map()` / `can_build_at()` / `build_tower_at()` / `free_slot_at()` / `get_slot_difficulty()`
- `data/maps/map_001.tres` — 初始地图数据（9 槽，含 difficulty）
- `data/maps/map_002.tres` — 第 2 张图数据模板（8 槽，中央蛇形路径）

### 创建新地图（在编辑器中）
1. 复制 `tower_defense.tscn` 为 `场景/levels/map_002.tscn`
2. 修改 TileMapLayer 瓦片布局 / EnemyPath 路径曲线
3. 调整槽位 Marker2D 坐标使其适配新路径
4. 用 Python 脚本重新计算每槽 dififculty 并写入 `map_002.tres`
5. 在 main.gd 中添加地图切换逻辑
