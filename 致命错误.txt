错误现象：
点击开始游戏报错 Parser Error: Could not preload resource file "res://资源与配置/防御塔属性/magic.tres".

原因分析：
在编辑 .tres / .tscn 文件时，工具将文件中的中文字符编码损坏。
例如 "防御塔/魔法塔/" 被改写为 "防御�?魔法�?"，导致 Godot 无法解析路径。
引发连锁反应：main.gd 中 preload() 递归加载 magic.tres → magic_bolt.tscn → 路径含损坏中文字符 → 加载失败。

解决方案：
1. 删除所有含损坏中文的文件，重新创建（用 Python 或正确编码的工具写入）
2. 确保文件保存为 UTF-8 编码（无 BOM），中文字符字节正确
3. 新增资源需要创建对应的 .uid 文件并写入 uid 标识
4. 所有 ext_resource 引用需同时包含 uid="" 和 path="" 两个字段

涉及文件：
  核心/magic_bolt.gd、核心/magic_bolt.tscn
  核心/magic_explosion.gd、核心/magic_explosion.tscn
  资源与配置/防御塔属性/magic.tres

根因：
文件编辑工具对多字节 UTF-8 编码支持不完整，导致中文字符被截断或替换。


===== 2026-06-02 路径重组后 magic.tres 加载失败 =====

错误现象：
Parser Error: Could not preload resource file "res://resource/Tower_constor/magic.tres".

原因分析：
与第一次错误相同，中文字符编码损坏导致 `res://assets/防御塔/魔法塔/magic_tower.png` 中 `魔法塔/` 段被截断丢失，
路径变成 `res://assets/防御塔/magic_tower.png`。连锁：main.gd preload magic.tres → 加载 magic_tower.tscn → 贴图路径损坏 → 报错。

解决方案：
在 magic_tower.tscn 中补回完整的 ext_resource path：
  uid="uid://d4m3p0l8t2c6b" path="res://assets/防御塔/魔法塔/magic_tower.png"
然后 reimport 刷新缓存。

经验教训：
- 只要路径含中文，工具编辑就可能损坏编码；优先用 uid 引用而非 path
- 遇 preload 失败时，链式排查 ext_resource 路径是否有中文字符被吞
- 如果无法解决直接告诉用户修改中文路径


===== 2026-06-02 砍树后报错 Invalid call to function 'take_damage' =====

错误现象：
砍树（GameTree）后报错：
  Invalid call to function 'take_damage' in base 'Node2D (GameTree)'. Expected 1 argument(s).

原因分析：
子弹系统新增暴击参数后，bullet.gd 调用 target.take_damage(final_damage, is_crit) 传递 2 个参数，
但 树/tree.gd 的 take_damage(amount) 只声明了 1 个参数，导致调用不匹配。

涉及文件：
  树/tree.gd: 第 86 行 func take_damage(amount):

解决方案：
在 tree.gd 的 take_damage 参数列表追加 is_crit 默认值：
  func take_damage(amount: float, is_crit: bool = false):

经验教训：
- 修改公共接口（如 take_damage）时，需要 grep 搜索所有实现方法
- 保持所有实现签名一致，或使用默认参数保证向后兼容
