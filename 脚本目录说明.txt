## 游戏流程 / Game Flow

```
1. Player clicks Start Wave → GameManager.start_wave()
2. GameManager spawns enemies via Timer
3. Enemy walks along Path2D → reaches end → lives--
4. Tower detects enemy in range → cooldown timer → _shoot()
5. Tower spawns bullet → bullet.initialize(target, damage)
6. Bullet moves toward target → Area2D collision → enemy.take_damage()
7. Enemy HP ≤ 0 → die() → gold += reward
8. Wave ends → player can place more towers
```

## 信号关系 / Signal Relationships

```
GameManager
├── gold_changed → main._update_gold()
├── lives_changed → main._update_lives()
├── wave_started → main._on_wave_started()
├── wave_done → main._on_wave_done()
├── enemy_killed → (bonus gold signal)
└── game_over → main._on_game_over()

Enemy
├── died → GameManager._on_enemy_died()
└── reached_end → GameManager._on_enemy_reached_end()

bullet (Area2D)
└── area_entered → _on_area_entered() → enemy.take_damage()
```

## 代码规范 / Code Conventions

1. **子弹初始化**：使用 `bullet.initialize(target, damage)` 而非直接设置属性
2. **preload 位置**：作为类级别变量（如 `tower_base.gd` 中的 `bullet_scene`），避免每帧重复加载
3. **类型标注**：使用 GDScript 4.x 的 `-> void` / `-> bool` 等返回类型标注
4. **信号连接**：在 `_ready()` 中连接，避免重复连接
5. **enemy group**：怪物统一添加 `"enemy"` group，便于 Area2D 检测
6. **私有变量**：内部状态用 `_` 前缀（如 `_last_pos`、`_spawn_timer`、`_CLICK_RADIUS_SQ`）
7. **性能优化**：
   - `distance_squared_to()` 替代 `distance_to()` 避免 sqrt
   - `maxf()` 替代 `max()` 避免类型推断
   - `slot.get_child_count() == 0` 替代遍历检查 tower group
   - `_enemy_path` 延迟获取，避免每波重复查找
8. **early return**：在 `_on_enemy_reached_end` 中 lives > 0 时先处理 game_over 分支再调 `_check_wave_done`

## 波次系统 / Wave System

### 架构概览 / Architecture

```
WaveConfigPanel.tscn （@tool 开发者面板 —— 独立场景，编辑器内打开配置）
        │
        │  点击 "导出" 按钮
        ▼
wave_config.tres （Resource 数据文件 —— GameManager 启动时加载）
        │
        ▼
GameManager.start_wave() （按 wave 索引读取对应 WaveEntry）
        │
        ▼
Timer._spawn_enemy() （按 entry 的 enemy_scene / count / spawn_interval 生成）
```

### 文件说明 / File Reference

| 文件 | 类型 | 职责 |
|------|------|------|
| `资源/wave_entry.gd` | Resource（class_name WaveEntry） | 单波配置数据：enemy_scene, count, spawn_interval |
| `场景/WaveConfigPanel.tscn` | Control（@tool） | 开发者配置面板，Inspector 编辑 waves 数组 |
| `场景/wave_config_panel.gd` | @tool 脚本 | 面板逻辑：导出到 .tres、运行时自动隐藏 |
| `配置/wave_config.tres` | Resource 文件 | 序列化的波次数据，GameManager 运行时读取 |

### WaveEntry 字段说明 / WaveEntry Fields

| 字段 | 类型 | 说明 |
|------|------|------|
| `enemy_scene` | PackedScene | 敌人场景（拖入 res://怪物/green_monster.tscn 等） |
| `count` | int | 该波次生成此敌人的数量 |
| `spawn_interval` | float | 每个敌人的生成间隔（秒） |

### GameManager 修改 / GameManager Changes

- `start_wave()`：从 `wave_config.tres` 按 wave 索引读取 WaveEntry，不再硬编码公式
- 第 1 波读取数组索引 0，第 5 波读取索引 4，依此类推
- 若配置不足，自动补齐默认值（green_monster × 8，间隔 0.5s）

### 添加新波次 / Adding New Waves

1. 打开 `场景/WaveConfigPanel.tscn`
2. 在 Inspector 中找到 `Waves` 数组
3. 增加数组元素，配置 enemy_scene / count / spawn_interval
4. 点击场景中的 **"导出配置"** 按钮
5. 运行游戏测试

### 添加新敌人类型 / Adding New Enemies

1. 创建新的敌人场景（继承 enemy_base.tscn）
2. 在 `WaveConfigPanel` 的 `Waves` 数组中，将 `enemy_scene` 指向新场景
3. 导出配置并运行

```

Wave 2 (future):
```

### 注意事项 / Notes

- `WaveConfigPanel` 是 **只供开发者使用** 的工具场景，运行时自动隐藏
- 配置完成后必须点击 **"导出配置"** 按钮，`wave_config.tres` 才会更新
- 导出后重新运行游戏即可生效，无需重启编辑器
